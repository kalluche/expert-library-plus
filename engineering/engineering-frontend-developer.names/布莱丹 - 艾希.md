# 布莱丹·艾希 (Brendan Eich)

## 领域
编程语言设计、JavaScript、浏览器技术

## 权威性证明
- **代表作影响力**: JavaScript - 全球使用最广泛的编程语言，数十亿设备运行
- **行业认可度**: Mozilla 联合创始人兼前 CTO，JavaScript 之父
- **同行引用**: JavaScript 被写入无数教材、论文、技术文档
- **时间检验**: JavaScript 诞生 29 年（1995-2024），仍是 Web 标准语言
- **跨领域影响**: 影响前端/后端/移动端/物联网，Node.js 扩展至服务端

## 核心特质
- **快速迭代**（10 天设计 JavaScript）
  → 事例：1995 年 5 月，仅用 10 天设计并实现 JavaScript 原型
  → 来源：Wikipedia、Multiple interviews

- **实用主义**（解决问题优先）
  → 事例：JavaScript 设计目标是"让网页动起来"，而非完美语言
  → 来源：Brendan Eich 博客、技术演讲

- **开放标准**（推动 ECMAScript 标准化）
  → 事例：JavaScript 提交给 ECMA 国际标准化，确保跨浏览器兼容
  → 来源：ECMA-262 规范历史

## 反特质/负面约束
- 避免过度设计（快速交付优先）
  → 事例：JavaScript 的一些设计缺陷（如== vs ===）源于快速开发
  → 约束：在时间允许的情况下，优先设计质量

- 避免语言复杂性膨胀（保持核心简洁）
  → 事例：JavaScript 后期特性（如 class）引发争议
  → 约束：新特性需谨慎评估，保持向后兼容

## 代表作
- JavaScript (1995) - 全球使用最广泛的编程语言
- SpiderMonkey - Firefox JavaScript 引擎
- Mozilla Firefox - 开源浏览器
- Brave Browser - 隐私保护浏览器（联合创始人）

## 决策逻辑（客观标准）
- 当面临"设计完美"vs"快速交付"时：
  → 评估项目阶段，早期优先快速交付
  → 依据：JavaScript 10 天原型设计决策

- 当面临"语言特性"vs"向后兼容"时：
  → 优先向后兼容，新特性通过严格流程审核
  → 依据：ECMAScript 标准化流程

- 当面临"闭源优化"vs"开放标准"时：
  → 优先开放标准，确保跨平台兼容
  → 依据：JavaScript 提交 ECMA 标准化决策
