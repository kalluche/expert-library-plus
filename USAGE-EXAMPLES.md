# 使用示例

## 🎯 基础使用流程

```
请专家帮我设计一个革命性的智能手表
```

**预期响应**:
```
收到！根据你的任务，推荐以下专家：

🎯 推荐专家列表：

1. **Product Manager** - 必需
   擅长：产品定义、用户体验、市场定位
   负责：产品愿景和功能规划

2. **UI Designer** - 推荐  
   擅长：界面设计、交互体验
   负责：手表界面和用户体验

3. **Frontend Developer** - 推荐
   擅长：前端实现、性能优化
   负责：技术可行性评估

请选择：
A) 使用全部专家
B) 自定义选择
C) 添加人名增强（推荐）

> 如果选择 C，系统会显示可用人名：
> - **史蒂夫·乔布斯** - 极致简约、产品直觉
> - **Jony Ive** - 工业设计、材料创新
> - **Tony Fadell** - 硬件产品、用户体验
```

## 🔧 故障排除

### 问题 1: 专家未被识别
**症状**: 用户说"请专家..."但系统没有响应专家流程

**解决方案**:
1. 确认文件已正确复制到 `~/.openclaw/experts/`
2. 检查目录结构是否包含所有子目录
3. 重启 OpenClaw 服务重新加载专家库
4. 验证命令: `ls ~/.openclaw/experts/engineering/`

### 问题 2: 人名库未显示
**症状**: 专家被识别但没有人名选项

**解决方案**:
1. 确认 `names/` 目录存在且包含 `.md` 文件
2. 检查人名文件格式是否正确（必须是 Markdown）
3. 验证路径: `~/.openclaw/experts/engineering/names/steve-jobs.md`

### 问题 3: 冲突检测不工作
**症状**: 系统没有提示人名冲突

**说明**: 冲突检测功能需要在后续版本中实现，当前版本主要提供人名选择功能。

## 📊 预期效果 vs 实际效果

### 预期效果:
- **质量提升**: AI 输出更具专业性和深度
- **风格一致**: 符合所选专家的思维模式
- **结果导向**: 聚焦于高质量成果而非过程描述

### 可能的实际情况:
- **效果因任务而异**: 复杂任务效果更明显，简单任务可能无显著差异
- **人名选择很重要**: 选择合适的人名对效果影响很大
- **需要调优**: 可能需要多次尝试找到最佳组合

## 💡 使用建议

1. **从简单任务开始**: 先测试基本功能
2. **记录对比结果**: 保存有/无人名增强的输出进行对比
3. **反馈改进**: 遇到问题及时提交 Issue
4. **社区协作**: 分享你的人名库和使用经验

---

**记住**: 这是一个实验性项目，效果可能因模型版本、任务类型和个人偏好而异。我们鼓励用户通过实际使用来验证和完善这个概念！
