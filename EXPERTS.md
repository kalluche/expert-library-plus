## ✏️ 自定义专家 (`custom/`)

> **用途：** 存放可复用的通用专家（手动添加）
> 
> **示例：**
> - 你有跨项目的技术栈偏好（如只用 Vue）
> - 你有行业专有知识（如医疗、金融）
> - 你有经验固化（踩过的坑不想再犯）
> 
> **不是：** 项目专属配置（项目配置记录到 PROJECT.md）

---

## 📁 关于项目记录

> **专家库不存储项目数据**。
> 
> 项目执行记录存储于立项目录：
> 
> ```
> projects/{任务名}_{YYYYMMDD-HHMM}/
> ├── PROJECT.md        # 项目知识库（含专家配置）
> ├── briefing.md       # 任务简报
> ├── logs/             # 执行日志
> └── final_output/     # 交付物
> ```
> 
> **项目专属的专家配置**（如技术栈偏好、特定需求）记录在 `PROJECT.md` 中，不是创建新专家文件。

## 🎯 使用方式

### 自然语言触发

```
"请专家帮我开发一个 Chrome 扩展"
"需要专家处理这个 API 设计"
"这个任务需要安全专家审查"
```

### 专家选择流程

1. 你说出需求
2. 我推荐专家列表（宁多勿少）
3. 你确认/调整
4. 我加载角色卡
5. 任务分析 + 需求澄清
6. 自动立项 + 执行

## 📋 专家角色卡结构

每个专家角色卡包含：

```markdown
# 🎯 专家名称

## 身份认知
- 你是谁
- 专业背景

## 核心使命
- 你负责什么
- 产出什么

## 关键规则
- 必须遵守的最佳实践
- 绝对不能做的事

## 工作流程
1. 第一步做什么
2. 第二步做什么
...

## 交付物标准
- 必须包含的内容
- 质量指标

## 沟通风格
- 如何与用户交流

## 常见陷阱
- 需要避免的错误
```
