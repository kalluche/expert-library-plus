# 🎭 专家系统使用说明

---

## 🚀 快速开始

### 触发专家系统

对 OpenClaw 说：

```
"请专家帮我 XXX"
"需要专家处理 XXX"
"这个任务需要专家"
"调用 XXX 专家"
```

### 示例

```
你： "请专家帮我开发一个 Chrome 扩展，监控网站价格变化"

我： "收到！根据你的任务，推荐以下专家：

🎯 推荐专家列表：

1. **Frontend Developer** - 必需
   擅长：Chrome 扩展开发、UI 实现
   负责：扩展界面、内容脚本

2. **Backend Architect** - 推荐
   擅长：API 设计、数据架构
   负责：价格数据存储、API 设计

3. **Security Engineer** - 推荐
   擅长：扩展安全审查
   负责：权限审查、数据安全

4. **Code Reviewer** - 可选
   擅长：代码质量审查
   负责：最终代码审查

请选择：
A) 用全部 4 个专家（最全面）
B) 用前 3 个专家（推荐）
C) 自定义选择（告诉我用哪几个）
D) 添加其他专家（告诉我加什么）"

你： "用前 3 个专家"

我： "好的！激活以下专家：
✅ Frontend Developer
✅ Backend Architect
✅ Security Engineer

现在开始任务分析..."
```

## 📋 完整工作流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 触发："请专家帮我 XXX"                              │
├─────────────────────────────────────────────────────────┤
│  2. 推荐：我分析任务 → 推荐专家列表（宁多勿少）          │
├─────────────────────────────────────────────────────────┤
│  3. 选择：你确认/调整专家                               │
├─────────────────────────────────────────────────────────┤
│  4. 加载：我读取专家角色卡 → 注入系统提示                │
├─────────────────────────────────────────────────────────┤
│  5. 分析：专家给出详细规划                              │
│     - 任务拆解                                          │
│     - 交付物清单                                        │
│     - 时间预估                                          │
├─────────────────────────────────────────────────────────┤
│  6. 澄清：我问 N 个问题直到完全搞懂需求                  │
├─────────────────────────────────────────────────────────┤
│  7. 立项：自动创建项目记录                              │
│     projects/{任务名}_{YYYYMMDD-HHMM}/                  │
├─────────────────────────────────────────────────────────┤
│  8. 执行：专家按规划工作                                │
├─────────────────────────────────────────────────────────┤
│  9. 记录：每个步骤记录到项目文件                        │
├─────────────────────────────────────────────────────────┤
│  10. 交付：完成任务 → 交付成果 + 完整记录               │
└─────────────────────────────────────────────────────────┘
```

## 🎯 需求澄清示例

```
我（专家组合）： "收到！我们已分析任务，给出以下规划：

## 任务规划

### 阶段 1: 需求分析（当前）
### 阶段 2: 开发实现
### 阶段 3: 测试发布

## 需求澄清（请回答以下问题）

在开始之前，我们需要确认以下问题：

1. **监控网站范围**
   - 只监控特定网站（如淘宝、京东）？
   - 还是允许用户自定义添加任意网站？

2. **价格变化阈值**
   - 降价多少百分比时通知？（默认 10%）
   - 涨价是否需要通知？

3. **通知方式**
   - 浏览器通知？
   - 邮件通知？
   - 其他方式？

4. **数据存储**
   - 本地存储（仅本机可用）？
   - 云端同步（多设备共享）？

5. **其他功能**
   - 价格历史图表？
   - 多商品对比？
   - 降价预测？

请回答以上问题，我们会根据你的答案完善规划。
如有不清楚的地方，我们会继续追问，直到完全理解需求。"
```

## 📁 项目记录格式

### PROJECT.md（扩展版）

**专家配置说明：**
> 专家库是静态的，不存储项目数据。
> 项目专属的专家配置（技术栈、特定需求等）记录在 PROJECT.md 中。

```markdown
# 项目名称

## 📋 项目背景
- 目标
- 成功标准

## 🎭 激活专家

### 通用专家（来自专家库）
- Frontend Developer
- Backend Architect
- Security Engineer

### 项目专属配置 ⭐
> 这些不是新专家，而是对通用专家的**项目特定要求**

#### Frontend Developer - 项目特定要求
- 技术栈：Vue 3 + TypeScript
- 目标平台：Chrome 扩展（Manifest V3）
- 特定需求：支持 aish888, sjs47 论坛

#### Security Engineer - 项目特定要求
- 必须处理 Cookie 安全存储
- 必须处理验证码（预留接口）

## 📝 需求澄清记录
### Q1: 监控哪些网站？
A: 淘宝、京东、拼多多

### Q2: 价格变化阈值？
A: 降价超过 10% 时通知

## 📝 关键决策
- 决策记录表

## 🧠 经验教训
- 成功经验
- 踩过的坑

## 📊 项目状态
- 已完成
- 进行中
- 待开始

## 📝 任务日志
### {日期} {时间} - {任务名称}
- 任务：简要描述
- 负责专家：专家名
- 任务目标：具体目标
- 输出文件：文件路径
- 状态：⏳ / ✅ / ❌
```

## 🤝 多专家协作模式

### 模式 1：顺序协作

```
任务：开发一个完整的 API

1. Backend Architect → 设计架构
2. Security Engineer → 安全审查
3. Code Reviewer → 代码审查
4. DevOps Automator → 部署方案
```

### 模式 2：并行协作

```
任务：全面审查一个项目

Backend Architect → 架构审查
Security Engineer → 安全审查  → 汇总报告
Code Reviewer → 代码审查
```

## ⚠️ 注意事项

1. **专家不需要常驻** - 按需激活，任务完成后恢复通用模式
2. **需求澄清很重要** - 问 N 个问题直到完全搞懂，避免返工
3. **全程记录** - 每个步骤都记录到项目文件，保证可追溯
4. **立项记录** - 重要任务自动立项，参照现有立项流程

## 🔧 技术实现

### 专家库位置

```
~/.openclaw/experts/
├── EXPERTS.md              # 专家索引
├── USAGE.md                # 使用说明（本文件）
├── engineering/            # 工程类专家
├── design/                 # 设计类专家
├── business/               # 商业类专家
├── safety/                 # 安全类专家
├── custom/                 # 自定义专家
└── sessions/               # 会话记录
```

### 角色卡加载

```
1. 读取专家 Markdown 文件
2. 注入到系统提示
3. 代理扮演专家角色
4. 任务完成后恢复
```
